Installing **Grafana Tempo** natively (without Docker) is pretty straightforward if you treat it like a single binary service.

I’ll walk you through a clean, production-style setup on Linux (Ubuntu/Debian). Same idea works on other distros.

---

## 1️⃣ Download Tempo binary



Go to the official releases page:

* [Grafana Tempo releases](https://github.com/grafana/tempo/releases?utm_source=chatgpt.com)

Then on your server:

```bash
wget https://github.com/grafana/tempo/releases/latest/download/tempo-linux-amd64.tar.gz
tar -xvzf tempo-linux-amd64.tar.gz
cd tempo-linux-amd64
```

Make it executable:

```bash
chmod +x tempo
```

Move it to a standard path:

```bash
sudo mv tempo /usr/local/bin/
```

---

## 2️⃣ Create config file

Create a config file:

```bash
nano tempo.yaml
```

Minimal working config:

```yaml
server:
  http_listen_port: 3200

distributor:
  receivers:
    otlp:
      protocols:
        http:
        grpc:

ingester:
  trace_idle_period: 10s
  max_block_bytes: 1_000_000
  max_block_duration: 5m

compactor:
  compaction:
    block_retention: 24h

storage:
  trace:
    backend: local
    local:
      path: /var/tempo/traces
```

Create storage directory:

```bash
sudo mkdir -p /var/tempo/traces
sudo chmod -R 777 /var/tempo
```

---

## 3️⃣ Run Tempo

```bash
tempo -config.file=tempo.yaml
```

If everything is fine, it starts listening on:

* HTTP → `localhost:3200`
* OTLP → ports for traces ingestion

---

## 4️⃣ Run as a systemd service (recommended)

Create service file:

```bash
sudo nano /etc/systemd/system/tempo.service
```

Paste:

```ini
[Unit]
Description=Grafana Tempo
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/tempo -config.file=/etc/tempo.yaml
Restart=always

[Install]
WantedBy=multi-user.target
```

Move config:

```bash
sudo mv tempo.yaml /etc/tempo.yaml
```

Start service:

```bash
sudo systemctl daemon-reexec
sudo systemctl daemon-reload
sudo systemctl enable tempo
sudo systemctl start tempo
```

Check:

```bash
sudo systemctl status tempo
```

---

## 5️⃣ Connect to Grafana



In Grafana:

* Go to **Data Sources**
* Add **Tempo**
* URL:

  ```
  http://localhost:3200
  ```

---

## 6️⃣ Send traces (important)

Tempo does **not generate traces itself** — you must send them from apps using:

* OpenTelemetry
* Jaeger / Zipkin compatible clients

Example OTLP endpoint:

```
http://<your-server>:4318
```
